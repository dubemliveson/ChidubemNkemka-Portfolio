Azure Resource Cleaner

During my time at the Stanbic Disep 4.0 tech program, my track was focused on C# and Azure where I and my team developed an Azure Resource Cleaner designed to automatically delete unused cloud resources using tags called "Test."
This application ensures cost optimization and reduces clutter in cloud environments. To enhance security and prevent unauthorized access, I integrated the Microsoft Authentication Library (MSAL) , which restricts access to authorized personnel only, thereby preventing accidental deletions. The application cannot be executed without a valid Microsoft email login, adding an essential layer of authentication.

Key tools and services utilized include:
- Azure Function App : Configured to trigger the cleanup process on a scheduled basis (e.g., every 2 days or weeks or months as the case may be). For demonstration purposes, the interval was set to 2 minutes to showcase its functionality.

- Application Insights : This was implemented for extensive logging and real-time monitoring which enabled us to carry out detailed tracking of the applicationâ€™s operations. This allowed for efficient troubleshooting and auditability by maintaining a historical record of all processes.

- C# : This is the programming language used to develop the application with the integration of the .NET framework.

- Azure : This was the cloud service provider used. It contained all the cloud resources used (storage account, application insights, function app, Microsoft Entra ID) to test the functionality of the application.

This project highlights my ability to combine cloud automation, security best practices, and monitoring tools to deliver a robust, scalable solution for resource management in Azure.
